# Data Dictionary

This document defines the schema, grain, relationships, and field semantics for the synthetic healthcare operations dataset generated by this repository.

## Dataset Metadata

- Data is synthetic (non-PHI) and reproducible when using the same `SEED` setting in `config.py`.
- Default date window is a rolling 3 years: `START_DATE = today - 1095 days`, `END_DATE = today`.
- Output location: `synthetic_healthcare_data/`.

## Table Relationships

- `patients.csv` is the patient dimension (`patient_id` is unique).
- `providers.csv` is the provider dimension (`provider_id` is unique).
- `appointments.csv` is an appointment event fact table with one row per room/status event.
- `treatments.csv` is a treatment-line fact table with one row per patient line-of-therapy.
- `date_dim.csv` is a calendar dimension with one row per calendar date.

Join paths:

- `appointments.mrn` -> `patients.mrn`
- `appointments.provider_id` -> `providers.provider_id`
- `treatments.patient_id` -> `patients.patient_id`
- `appointments.appointment_date` -> `date_dim.date`
- `treatments.start_date` / `treatments.end_date` -> `date_dim.date`

Important grain notes:

- `appointments.csv` is not one row per appointment. A completed visit has multiple rows with the same `appointment_id` (one row per flow step).
- For appointment-level metrics, deduplicate by `appointment_id` (or use one row per `appointment_id` after aggregation).
- `treatments.csv` can have multiple rows per patient (`line_of_therapy` 1..N).

## patients.csv

Table purpose: Patient-level demographics, diagnosis profile, and high-level clinical flags.

Grain: One row per patient.

Primary key: `patient_id`

| Column | Type | Nullable | Description | Allowed values / range | Generation notes |
| --- | --- | --- | --- | --- | --- |
| patient_id | string | No | Patient surrogate identifier. | Format `PT` + 6 digits (for example `PT100001`). | Sequential during generation. |
| mrn | string | No | Medical record number (synthetic). | Format `MRN` + 6 digits. | Derived from patient index. |
| first_name | string | No | Synthetic first name. | Free text. | Sex-specific fake name generator. |
| last_name | string | No | Synthetic last name. | Free text. | Faker-generated surname. |
| sex | string | No | Recorded sex. | `Female`, `Male`. | Diagnosis-weighted logic (for example, Prostate -> Male, Ovarian -> Female). |
| dob | date | No | Date of birth. | Historical calendar date. | Generated by helper utility. |
| age | integer | No | Age in years at generation time. | Non-negative integer. | Computed from `dob`. |
| race | string | No | Race category. | `White`, `Black`, `Asian`, `Hispanic`, `Other`. | Weighted by `RACE_WEIGHTS` in `config.py`. |
| zip_code | string | No | ZIP code (synthetic). | 5-digit text. | Randomized plausible ZIP pattern. |
| insurance_type | string | No | Primary insurance category. | `Commercial`, `Medicare`, `Medicaid`, `Self-Pay`. | Age-dependent weighting (<65 vs >=65). |
| diagnosis_group | string | No | Broad diagnosis grouping. | `malignant`, `ida`. | Weighted by `DIAGNOSIS_GROUPS` (default 40%/60%). |
| primary_diagnosis | string | No | Primary diagnosis label. | 9 malignant diagnoses plus `Iron Deficiency Anemia`. | Derived from diagnosis group and weighted diagnosis mix. |
| stage | string | No | Staging or risk category. | Solid tumors: `I`,`II`,`III`,`IV`; heme cancers (`Lymphoma`,`Multiple Myeloma`): `Low Risk`,`Intermediate Risk`,`High Risk`; IDA: `N/A`. | Diagnosis-specific staging rule. |
| comorbidity_score | integer | No | Synthetic comorbidity burden score. | Integer, clipped to 0..6 (IDA) or 0..8 (malignant). | Poisson-distributed by diagnosis and age. |
| smoking_status | string | No | Tobacco-use status. | `Never`, `Former`, `Current`. | Lung cancer uses different weighting than non-lung diagnoses. |
| active_treatment_flag | integer | No | Whether patient is currently in treatment cohort. | `0` or `1`. | Group-specific probability (`ida` vs `malignant`). |
| deceased_flag | integer | No | Mortality indicator. | `0` or `1`. | Very low probability, lower in IDA group. |

## providers.csv

Table purpose: Provider roster used for appointment assignment and provider throughput analysis.

Grain: One row per provider.

Primary key: `provider_id`

| Column | Type | Nullable | Description | Allowed values / range | Generation notes |
| --- | --- | --- | --- | --- | --- |
| provider_id | string | No | Provider surrogate identifier. | Format `PR` + 4 digits (for example `PR1001`). | Sequential during generation. |
| provider_name | string | No | Provider display name. | Format `Dr. <First> <Last>`. | Faker-generated names. |
| years_experience | integer | No | Years in practice. | Integer clipped to 1..35. | Drawn from normal distribution (mean 12, sd 6). |
| fte | float | No | Full-time equivalent. | `1.0`, `0.8`, `0.6`. | Weighted 70/20/10. |

## appointments.csv

Table purpose: Appointment operations fact table with room-level event sequence and per-event timing deltas.

Grain: One row per room/status event within an appointment.

Natural keys:

- Appointment-level identifier: `appointment_id`

Foreign keys:

- `provider_id` -> `providers.provider_id`
- `appointment_date` -> `date_dim.date`

| Column | Type | Nullable | Description | Allowed values / range | Generation notes |
| --- | --- | --- | --- | --- | --- |
| appointment_id | string | No | Appointment identifier. | Format `APPT-` + 10 uppercase hex chars. | UUID-derived token. |
| mrn | string | No | Patient MRN snapshot. | Matches patient MRN format. | Copied from patient dimension. |
| provider_id | string | No | Assigned provider identifier. | Must exist in `providers.csv`. | Random provider sample per visit. |
| appointment_date | date | No | Scheduled visit date. | Calendar date in configured range. | Derived from scheduled visit datetime at generation time. |
| visit_type | string | No | Visit category. | `New Patient`, `Follow-up`, `Infusion`, `Lab Review`, `Urgent Visit`. | Diagnosis and treatment-state weighted logic. |
| status | string | No | Appointment status. | `Completed`, `No Show`, `Cancelled`. | Infusion has higher completion probability. |
| new_patient_flag | integer | No | New patient indicator. | `0` or `1`. | 1 when `visit_type` is `New Patient`. |
| infusion_flag | integer | No | Infusion visit indicator. | `0` or `1`. | 1 when `visit_type` is `Infusion`. |
| urgent_flag | integer | No | Urgent visit indicator. | `0` or `1`. | 1 when `visit_type` is `Urgent Visit`. |
| room | string | No | Room/state label for this event row. | Workflow state or physical room like `Exam Rm 3`, `Infusion Bay 8`. | Completed visits resolve generic states to physical rooms for exam/infusion. |
| room_type | string | No | Normalized room grouping. | `Front Desk`, `Lab`, `Waiting Room`, `Exam Room`, `Other`, `Status`. | Mapped from `room` using room-type logic. |
| room_timestamp | datetime | No | Event start timestamp for the row's room/state. | Timestamp. | For completed visits, increments through the generated room flow. |
| duration | float | Yes | Minutes between this row's `room_timestamp` and the previous room event within the same appointment. | Null for first room event in each appointment. | Computed as per-appointment timestamp diff ordered by `room_timestamp`. |

Operational caveats:

- For `Cancelled` and `No Show`, exactly one status row is created.
- `duration` is null for the first row of each appointment by design.

## treatments.csv

Table purpose: Longitudinal treatment-line records per patient.

Grain: One row per patient treatment line.

Primary key: `treatment_id`

Foreign keys:

- `patient_id` -> `patients.patient_id`
- `start_date` / `end_date` can be joined to `date_dim.date`

| Column | Type | Nullable | Description | Allowed values / range | Generation notes |
| --- | --- | --- | --- | --- | --- |
| treatment_id | string | No | Treatment-line identifier. | Format `TRT-` + 10 uppercase hex chars. | UUID-derived token. |
| patient_id | string | No | Patient identifier. | Must exist in `patients.csv`. | Inherited from patient row. |
| line_of_therapy | integer | No | Treatment line index. | Positive integer starting at 1. | Number of lines depends on stage/risk and active status. |
| regimen_name | string | No | Regimen/protocol name. | Diagnosis-specific values (for example `FOLFOX`, `R-CHOP`, `Venofer`, `Observation`). | Chosen from diagnosis-specific regimen dictionary. |
| treatment_category | string | No | Regimen class. | `Chemo`, `Targeted Therapy`, `Immunotherapy`, `Hormonal Therapy`, `Supportive Therapy`, `Observation`. | Derived from `regimen_name`. |
| route | string | No | Administration route. | `IV`, `Oral`, `Injection`, `None`. | Derived from treatment category mapping. |
| frequency | string | No | Administration frequency shorthand. | `Daily`, `Q1W`, `Q2W`, `Q3W`, `Q4W`, `Q3D`, `Q7D`, `Monthly`, `Q3M`, `Observation`. | Derived from category/route/regimen logic. |
| intent | string | No | Treatment intent. | `Curative`, `Maintenance`, `Palliative`, `Supportive`. | Stage/risk weighted for non-Venofer regimens. |
| start_date | date | No | Line start date. | On or after patient's first appointment date. | Line 1 starts near first appointment; later lines offset from prior line end. |
| end_date | date | No | Line end date. | `>= start_date`, clipped to global `END_DATE`. | Duration sampled by category and capped at `END_DATE`. |
| active_flag | integer | No | Whether line is active at `END_DATE`. | `0` or `1`. | 1 when `start_date <= END_DATE <= end_date` prior to end-date clipping. |

Clinical logic notes:

- IDA patients use `Venofer` or `Observation` with lower line counts.
- Stage IV and High Risk patients are more likely to have multiple lines and palliative intent.

## date_dim.csv

Table purpose: Calendar dimension for time-based reporting.

Grain: One row per date.

Primary key: `date`

| Column | Type | Nullable | Description | Allowed values / range | Generation notes |
| --- | --- | --- | --- | --- | --- |
| date | date | No | Calendar date key. | Daily dates from `START_DATE` to `END_DATE`, inclusive. | Built with daily date range. |
| year | integer | No | Calendar year. | 4-digit year. | Derived from `date`. |
| month | integer | No | Calendar month number. | 1..12. | Derived from `date`. |
| month_name | string | No | Calendar month name. | `January`..`December`. | Derived from `date`. |
| quarter | string | No | Quarter label. | `Q1`..`Q4`. | Derived from `date`. |
| day_of_week | string | No | Day name. | `Monday`..`Sunday`. | Derived from `date`. |
| week_of_year | integer | No | ISO week number. | 1..53. | Derived via ISO calendar week. |
| is_weekend | boolean | No | Weekend flag. | `True` or `False`. | True when day-of-week is Saturday or Sunday. |

## Suggested Querying Patterns

- Appointment count (unique visits): count distinct `appointment_id` from `appointments.csv`.
- Room utilization: aggregate `duration` by `room`, `room_type`, and time grain.
- Provider throughput: distinct appointments by `provider_id`, not raw row count.
- Treatment timeline: sort by (`patient_id`, `line_of_therapy`) and compare `start_date`/`end_date`.

## Versioning Note

Because generation is stochastic (though seeded), counts and specific values can change when configuration or code changes. Rebuild this dictionary if schema or generation logic is modified.
